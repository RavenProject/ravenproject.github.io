WThe RVN Wallet couldn't be authenticated on Android.

The RVN Wallet uses the hardware encryption of Android phones to secure each user's funds. In order to access this system, it requires the user to enter their device passcode or provide a fingerprint. A bug in some versions of Android can cause this authentication to not work, in which case the RVN Wallet can not access the user's wallet. This issue has been fixed in Android version 8 and above. If you are experiencing this problem, update to Android 8 or higher. If your phone is unable to use Android 8, you can try the following, however there are some risks: Make sure your paper key is written down. In the Android settings, remove all fingerprints and device authentication by setting the screen lock setting to "none". This will erase all data protected by the hardware encryption, including your RVN Wallet data. Other apps may be affected as well, so proceed with caution. Re-enable lock screen authentication and re-register your fingerprints for fingerprint authentication. Open the RVN Wallet and restore your wallet with the paper key. This solution might not work on all phones. We suggest users update to Android 8 as soon as possible.
